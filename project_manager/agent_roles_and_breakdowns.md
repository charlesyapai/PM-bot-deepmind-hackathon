We will be deploying this project in a multi-agent manner - where each agent has its own unique role and capability, and communications with each agent will be done through .md files, treated like emails where different agents will drop their responses in that agent's .md 'mailbox'. This means that each agent will have their own folder which acts as a mailbox for that agent. We will have the following agents:

1. Overall Project Manager: The agent who will be acting upon this prompt - the one who will be creating the different mailboxes and determining how the folder structure will be set up, how it will be tested, etc. This agent will also be hyper focused on the project requirements and make sure to ensure that the project is completed to the project specifications, as well as be the main communication point with me, the main coder.
2. The overall architecture designer: The agent who will be designing the overall architecture of the app, and will be responsible for ensuring that the app is designed in a way that fits the project requirements.
3. The UX designer: The agent who will be designing the overall user experience of the app, and will be responsible for ensuring that the app is designed in a way that fits the project requirements as well, will be in charge of also validating and creating project requirement features for the application, and updating the project manager on the progress of the project, and also the document detailing what the application will look like/should do.
4. The logic-AI designer: The agent who will be deciding and coding the way the application is to interact with the AI API, focusing on understanding and calling and testing calling the inputs and the outputs of the AI application. Also it will be overseeing the prompting logic and guardrails of the prompts each time it is to be fed. Also focuses on the prompt engineering aspect of the application that will allow the output of the AI to be processed and then used to update the application.
5. The backend software engineer: The agent who will be responsible for implementing the backend of the app, focusing on the data being processed and updated.
6. The frontend software engineer: The agent who will be responsible for implementing the frontend of the app, focusing on the user interface and user experience.
7. The test engineer: The agent who will be responsible for testing the app, ensuring that it is working as intended and that it is free of bugs.
8. The voice-AI engineer: The agent who will be responsible for implementing the voice AI of the app, focusing on the voice AI and its integration with the app.
9. The documentation engineer: The agent who will be responsible for documenting the app, ensuring that the documentation is clear and easy to understand.
10. The senior engineer/version controller: The agent who will be responsible for overseeing the entire project like the project manager, but will be in charge of naming and shipping the github versions in terminal.

Note that each agent will be managed in a seperate prompt via the agent manager, and is also expected to update a progress report of their work in their own respective mailboxes so everyone can see their progress.
